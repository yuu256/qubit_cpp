## 1. ビット反転符号の仕組みと状態遷移

量子エラー訂正（ビット反転符号）では、1つのデータ量子ビット（$Q_0$）の情報を2つのデータ量子ビット（$Q_1, Q_2$）にコピー（符号化）して「多数決」が取れる状態を作ります 。さらに、重ね合わせを壊さないように補助ビット（$A_0, A_1$）を間接的に測定してエラーの場所（シンドローム）を特定します 。

シミュレータ内部では、状態ベクトル（配列のインデックス）が以下のように遷移していきます 。今回は量子ビットのインデックス配置を `Q0 Q1 Q2 A0 A1` の順（左が上位ビット）とします 。

1. **初期状態の設定**
データビット $Q_0$ に未知の量子状態をセットします（補助ビットは `0` 固定） 。



$$\lvert \psi \rangle = \alpha \lvert 00000 \rangle + \beta \lvert 10000 \rangle$$


配列上では `state[0]` に $\alpha$、`state[16]` に $\beta$ が入ります 。


2. **符号化（Encoding）**


$Q_0$ をコントロールにして $Q_1, Q_2$ へCNOTをかけることで、以下の絡み合い状態（GHZ状態）を作ります 。



$$\alpha \lvert 00000 \rangle + \beta \lvert 11100 \rangle$$


この時、$\beta$ の振幅は `state[16]` から `state[28]`（$\lvert 11100 \rangle$）へ引っ越します 。


3. **エラー発生（Noise）とシンドローム測定**
データビットのどこか1つ（今回は $Q_1$）にビット反転エラー（Xゲート）が発生したと仮定します 。
状態は $\alpha \lvert 01000 \rangle + \beta \lvert 10100 \rangle$ へと変化します 。
その後、補助ビット $A_0$ に $Q_0 \oplus Q_1$、$A_1$ に $Q_1 \oplus Q_2$ のパリティ情報を書き込み、$A_0$ と $A_1$ を測定します 。データを直接測定しないため、重ね合わせを壊さずにエラー位置を特定できます 。

| 測定結果 $(m_1, m_2)$ | 異常の検知内容 | エラーの場所 | 対策（Correction） |
| --- | --- | --- | --- |
| $(1, 0)$ | <br>$Q_0$ と $Q_1$ が異なる / $Q_1$ と $Q_2$ は同じ 

 | <br>$Q_0$ 

 | <br>`apply_X(state, Q0)` 

 |
| $(1, 1)$ | <br>$Q_0$ と $Q_1$ が異なる / $Q_1$ と $Q_2$ も違う 

 | <br>$Q_1$ 

 | <br>`apply_X(state, Q1)` 

 |
| $(0, 1)$ | <br>$Q_0$ と $Q_1$ は同じ / $Q_1$ と $Q_2$ が違う 

 | <br>$Q_2$ 

 | <br>`apply_X(state, Q2)` 

 |
| $(0, 0)$ | 誰も矛盾していない 

 | なし

## 2. 初期案と改善案の比較：陥りがちな2つの罠

下書き段階の初期案から、シミュレータを正しく動作させるために修正した2つのポイントを比較します 。

### 罠①：ストーリー（エラー発生箇所とシンドローム）の矛盾

* 
**初期案**： `apply_X(state, Q0);` （$Q_0$ にエラーを発生させている） 


* 
**改善案**： `apply_X(state, Q1);` （$Q_1$ にエラーを発生させる） 


* **比較と理由**：
初期案のコードでは $Q_0$ にエラーを適用していましたが、その後のシンドローム判定（`if` 文）やコメントでは「$Q_1$ のエラーを検知して直す流れ（`m1==1 && m2==1`）」を想定していました 。これではストーリーが矛盾し、エラー訂正が正常に機能しません 。エラー発生箇所を $Q_1$ に統一することで、シンドローム測定結果が $(m_1, m_2) = (1, 1)$ となり、綺麗に $Q_1$ が救済される整合性を確保しました 。



### 罠②：【重要】デコード（検証）時における CNOT の向き

* 
**初期案**： `apply_CNOT(state, Q1, Q0);` （$Q_1$ がコントロール、$Q_0$ がターゲット） 


* 
**改善案**： `apply_CNOT(state, Q0, Q1);` （$Q_0$ がコントロール、$Q_1$ がターゲット） 


* **比較と理由**：
符号化（Encoding）の際、私たちは「$Q_0$ を親（コントロールビット）」として $Q_1, Q_2$ へ状態をコピーしました 。したがって、エラーを直した後にデコード（逆操作）をして元の $Q_0$ に状態を一本化するときも、「$Q_0$ を親（コントロールビット）」としてCNOTをかけるのが正解です 。
もし初期案のように向きを逆にしてしまうと、$\alpha \lvert 00000 \rangle + \beta \lvert 11100 \rangle$ の状態から $\beta$ 側の振幅が元の `state[16]` ($\lvert 10000 \rangle$) に綺麗に戻らず、別のインデックスに迷子になってしまいます 。

## 3. 測定後の落とし穴：出力が `(0,0)` になる物理現象の怪

正しくエラー訂正とデコードまで終えた後、検証のために `state[0]` と `state[16]` を出力すると、値が `(0,0)` になってしまい「バグったのか？」と錯覚する現象にぶつかります 。

> **💡 原因はバグではなく、物理現象（状態の収縮）の完璧な再現**
> 今回のエラーシナリオ（$Q_1$ のエラー）では、シンドローム測定の結果、補助ビット $A_0$ と $A_1$ は両方とも `1` に確定します（`m1 = 1, m2 = 1`） 。
> Vol.3 で実装した通り、測定関数 `measure_qubit` は「選ばれなかった方の振幅をすべて `0` にする」という処理を行います 。
> 
> 
> ビット配置が `Q0 Q1 Q2 A0 A1` であるため、補助ビットが `11` に確定した瞬間、**末尾2ビットが `11` ではない項（＝補助ビットが `00` などの項）の振幅はすべて消滅して `0` になります** 。
> * `state[0]` （2進数 `00000`） $\rightarrow$ 補助ビットが `00` なので消滅（`0` に収縮） * `state[16]`（2進数 `10000`） $\rightarrow$ 補助ビットが `00` なので消滅（`0` に収縮） 
> 
> 

### 元のデータ（$\alpha, \beta$）はどこへ消えた？

消えたわけではなく、**補助ビットが `11` になった部屋へ引っ越しています** 。2進数の末尾に `11` がつくということは、10進数でいうと `+3` された位置にいるということです 。

* 
$\alpha$ の引っ越し先： `state[0 + 3]` $\rightarrow$ **`state[3]`** （`00011`） 


* 
$\beta$ の引っ越し先： `state[16 + 3]` $\rightarrow$ **`state[19]`** （`10111`） 



そのため、検証用の出力部分では「測定結果（`m1, m2`）に応じた正しいインデックス」を計算して覗いてあげる必要があります 。

## 4. 清書したメイン処理の完全なC++コード

上記の矛盾やデコードのバグをすべて修正し、コンソールの文字化け対策として日本語メッセージを英数字（英語）に変更した完全なメイン処理のコードです 。

```cpp
#include <iostream>
#include <vector>
#include <complex>

using Complex = std::complex<double>;
using StateVector = std::vector<Complex>;

// 量子ビットのインデックス定義 (左端がQ0)
const int Q0 = 0;
const int Q1 = 1;
const int Q2 = 2;
const int A0 = 3;
const int A1 = 4;

const int NUM_QUBITS = 5;
const int STATE_SIZE = 1 << NUM_QUBITS; // 32

int main() {
    // 状態ベクトルの初期化（全要素0）
    StateVector state(STATE_SIZE, 0.0);
    
    // 1. 初期状態の設定 (|ψ> = alpha|00000> + beta|10000>)
    double alpha = 0.6;
    double beta = 0.8;
    state[0] = alpha;  // |00000>
    state[16] = beta; // |10000> (1 << 4)
    
    std::cout << "--- 3-Qubit Bit-Flip Code Simulation ---" << std::endl;
    
    // 2. 符号化 (Encoding) : Q0 の状態を Q1, Q2 にコピー
    apply_CNOT(state, Q0, Q1);
    apply_CNOT(state, Q0, Q2);
    
    // 3. エラー発生 (Noise) : 今回は Q1 にビット反転エラーが発生したと仮定
    apply_X(state, Q1); 
    
    // 4. シンドローム測定の仕込み : 補助ビット A0, A1 にパリティを書き込む
    apply_CNOT(state, Q0, A0);
    apply_CNOT(state, Q1, A0);
    apply_CNOT(state, Q1, A1);
    apply_CNOT(state, Q2, A1);
    
    // 5. 測定の実行 : 補助ビット A0, A1 の測定結果(m1, m2)を取得
    int m1 = measure_qubit(state, A0); // Q1エラーの場合 = 1
    int m2 = measure_qubit(state, A1); // Q1エラーの場合 = 1
    std::cout << "Syndrome Measured: (m1, m2) = (" << m1 << ", " << m2 << ")" << std::endl;
    
    // 6. 復元 (Correction) : 古典ビットの測定結果に応じて if 文で分岐し、エラーを訂正
    if (m1 == 1 && m2 == 0) {
        apply_X(state, Q0);
    }
    else if (m1 == 1 && m2 == 1) {
        apply_X(state, Q1); // 見事 Q1 のエラーが救済される
    }
    else if (m1 == 0 && m2 == 1) {
        apply_X(state, Q2);
    }
    
    // 7. 検証 (Decoding) : 符号化の逆操作（Q0を親にしてCNOT）
    apply_CNOT(state, Q0, Q1);
    apply_CNOT(state, Q0, Q2);
    
    // 8. 結果の出力 : 状態収縮によって移動した先のインデックスを計算して確認
    int final_base_index = (m1 << 1) | m2; // 測定結果から補助ビットのズレ分を計算
    int idx_alpha = 0 + final_base_index;  // alphaの生存している部屋 (Q0=0)
    int idx_beta  = 16 + final_base_index; // betaの生存している部屋  (Q0=1)
    
    std::cout << "--- Final Result Verification ---" << std::endl;
    std::cout << "state[" << idx_alpha << "] (alpha): " << state[idx_alpha] << " (Expected: 0.6)" << std::endl;
    std::cout << "state[" << idx_beta << "] (beta) : " << state[idx_beta] << " (Expected: 0.8)" << std::endl;
    
    return 0;
}

```
