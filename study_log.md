## 1. C++における10進数から2進数への変換方法

C++で10進数の数値を2進数表現の文字列に変換する場合、用途や開発環境に合わせていくつかのアプローチを選択できます 。

* 
**`std::bitset` を使う方法（おすすめ）** 


* 標準ライブラリの `<bitset>` を使用する、最も手軽な方法です 。


* 
`std::bitset<8>(index)` のように `<桁数>` を指定して使用し、指定したビット幅の2進数表現に自動で変換されます 。


* 固定桁数（8桁や32桁など）でゼロ埋めして出力したい場合に最適です 。




* 
**`std::format` を使う方法（C++20以降）** 


* C++20以降の環境であれば、文字列フォーマット機能を利用してスマートに2進数表記（`{:b}` など）へ変換可能です 。




* 
**2で割り算を繰り返す方法（アルゴリズムの自作）** 


* ライブラリに頼らず、10進数を2で割り続けた「余り」を逆順に並べるという数学的な計算を愚直にコードにする方法です 。


* 頭の余分な「0」を省いた、純粋な計算結果としての2進数文字列が欲しい場合に適しています 。

## 2. 量子シミュレータにおける特定のビット値の抽出

量子シミュレータの内部処理（制御ゲートの実装など）では、「あるインデックスの特定のビットの値（0か1）を抽出する」というビット演算が頻出します 。

### 定番のビット演算式

```cpp
// index のなかの「target番目の量子ビット」の値（0か1）を取り出す式
// ※最上位ビットをQ0、最下位ビットをQNとする
int bit_val = (index >> (NUM_QUBITS - 1 - target)) & 1;

```

### 処理の2つのステップ

1. 
**右シフト (`>>`)**：目的のビット（`target`）を、一番右端（最下位ビット）まで移動させます 。


2. 
**論理積 (`& 1`)**：右端に持ってきたビット以外をすべて `0` にクリアし、そのビットが `0` か `1` かを確定させます 。



### 各パーツの詳細解説

* 
**`NUM_QUBITS - 1 - target` （シフト量の計算）** 


* 今回は、一般的なビットの数え方（右端が0番目）とは異なり、「左端（最上位）が0番目（Q0）」というルールを想定しています 。


* そのため、全量子ビット数（`NUM_QUBITS`）から「右端から数えて何桁目にあるか」をこの式で算出します 。


* 
*例*：`NUM_QUBITS = 4` で `target = 1` (Q1) の場合、右端からの距離は $4 - 1 - 1 = 2$ となります 。




* 
**`index >> (...)` （右シフト演算）** 


* 調べたいビットを一番右端の位置まで引きずり出す操作です 。


* 
*例*：`index = 10`（2進数で `1010`）のとき、`target = 1`（左から2番目の `0`）を調べたい場合、右に2回シフト（`>> 2`）して右端に `0` を持ってきます 。




* 
**`& 1` （マスク処理・抽出）** 


* 右シフトした後の値と `1`（2進数で `0001`）の論理積（AND演算）をとります 。


* AND演算は「両方とも 1 のところだけが 1 になる」ため、右端以外の数字は強制的に `0` にクリアされ、目的のビット値だけがピンポイントで抽出できます 。





$$\text{bit\_val} = (\text{index} \gg \text{右端からの距離}) \ \& \ 1$$

## 3. 量子Xゲート（NOTゲート）の正しい認識と実装

量子シミュレータにおいて、「Xゲート＝単なるビット反転」という古典コンピュータの感覚のままだと、コードの実装意図（なぜスワップが必要なのか）が掴めなくなります 。

### 古典コンピュータと量子シミュレータの違い

| 項目 | 古典コンピュータ（NOT） | 量子シミュレータ（Xゲート） |
| --- | --- | --- |
| **保持しているデータ** | メモリの中に `0` または `1` というデータが**直接**入っている 。

 | 0や1そのものではなく、すべての状態（可能性）がどれくらいの確率で存在するかという**「振幅（確率の波）」**を配列（`state`）に保持している 。

 |
| **ゲートの処理内容** | データそのものを直接 `0` から `1`、`1` から `0` へ書き換える 。

 | <br>`target`ビットが `0` である状態の振幅と、`1` である状態の**振幅を丸ごと入れ替える（スワップする）** 。

 |

### 具体例：2量子ビットでターゲットがQ0（左から1番目）の場合

Xゲートを適用すると、すべての状態のQ0ビットが反転するため、確率の波が以下のように移動します 。

* 
`00` という状態の振幅 $\rightarrow$ `10` の位置へ移動 


* 
`01` という状態の振幅 $\rightarrow$ `11` の位置へ移動 



シミュレータの内部視点では、これは `state[00]` と `state[10]`、`state[01]` と `state[11]` の中身を入れ替える **配列要素のスワップ（Swap）処理** になります 。

> **⚠️ 認識のズレに注意**
> * **NGな認識**：`index` 変数自体のビットを直接書き換えて終わりにする（これでは配列の中身の振幅が動かないため誤り） 。
> * **OKな認識**：`index` のビットを操作して「ペアになる相手の部屋番号（インデックス）」を特定し、**配列の要素（振幅）同士を `std::swap` する** 。
> 
> 

### 2重スワップを防ぐための実装アプローチ

0から31まで（5量子ビットの場合）愚直にループを回して、対象を見つけるたびにスワップしてしまうと、「0の項と1の項を入れ替えた後、ループが進んで1の項に到達したときに、また元に戻してしまう」というバグ（2重スワップ）が発生します 。これを防ぐアプローチは主に2つあります。

1. 
**最初から target ビットが0であるインデックスだけを狙ってループを回す** 


* ビット演算を使って、対象となるインデックス（`0`の項）を効率よく作り出す方法です 。無駄なループや条件分岐を減らせるため、シミュレータ実装において非常に推奨されるテクニックです 。




2. 
**0〜31までループを回し、条件分岐を入れる** 


* ループの中で「今のインデックスの target ビットが0のときだけ処理する（1のときはスルーする）」という条件分岐を挟みます 。この判定の際に、前述した「ビット抽出の式」がそのまま役に立ちます 。

# 【C++×量子計算】学習ログ Vol.2：1重ループによるCNOTゲートの実装

C++を用いた量子計算シミュレータの実装を学ぶ過程の学習ログ第2弾です 。前回のVol.1（ビット操作とXゲートの基礎）を踏まえ、今回は2つの量子ビット間で相互作用を行う「CNOTゲート（制御NOTゲート）」の本質的な概念と、シミュレータにおける効率的な実装アプローチについてまとめます 。

---

## 1. CNOTゲートのシミュレータにおける概念

CNOT（Controlled-NOT）ゲートは、2つの量子ビット（**controlビット**と**targetビット**）を対象とするゲートです 。古典計算の `if` 文（条件分岐）に似ていますが、状態ベクトルを扱うシミュレータの内部視点では、やはり「振幅（確率の波）のスワップ処理」として捉える必要があります 。

### 古典計算の直感とシミュレータの処理の対応

* 
**古典的なルール**：「controlビットが `1` の項だけ、targetビットの `0` と `1` を入れ替える」 


* 
**シミュレータ内部の処理**：「**controlビットが `1**`」かつ「**targetビットが `0**`」であるインデックス（部屋番号）を見つけ、それに対応する「targetビットが `1` のインデックス」と**中身（振幅）をスワップする** 



Xゲートがすべての項の `0` と `1` をひっくり返したのに対し、CNOTゲートは「controlビットが `1` である世界（状態）」にだけピンポイントでXゲート（スワップ）を適用する操作になります 。

---

## 2. なぜ「2重ループ」ではなく「1重ループ」で良いのか

「2つのビットをチェックする」と考えると、直感的にループを2重に回したくなりますが、状態ベクトル（`StateVector`）の配列を走査する際は「0〜31（5量子ビットの場合）を巡回する1重ループ」で完全に処理できます 。

1回のループ（ある1つのインデックス `index`）の中で、Vol.1で学んだ「ビット抽出式」を使い、**1つのインデックスから2つのビットの値を同時に取り出す**というアプローチを取るためです 。

### 1回分のループの思考フロー

1. ループで現在の `index` を確認 


2. 
`index` の `control` 番目のビットが `0` か `1` かを調べる 


3. 
`index` の `target` 番目のビットが `0` か `1` かを調べる 


4. 条件（`control == 1` かつ `target == 0`）を満たしていたら、ペアを探してスワップする 



---

## 3. CNOTゲートの具体的なC++実装

TODOの指示（0〜31までループ）通りに、ビット抽出式とビット和を用いて実装した具体的なコード例です 。

```cpp
// CNOTゲート：controlビットが1の項だけ、targetビットの0と1を入れ替える
void apply_CNOT(StateVector& state, int control, int target) {
    // 5量子ビットの場合（状態数は32通り）
    for (int index = 0; index < 32; ++index) {
        // 1. controlビットとtargetビットの値をそれぞれ抽出
        int control_val = (index >> (NUM_QUBITS - 1 - control)) & 1;
        int target_val  = (index >> (NUM_QUBITS - 1 - target)) & 1;
        
        // 2. 「controlが1」かつ「targetが0」の項だけを処理する（2重スワップ防止）
        if (control_val == 1 && target_val == 0) {
            // 3. targetビットを1に反転させたペアのインデックスを計算
            int pair_index = index | (1 << (NUM_QUBITS - 1 - target));
            
            // 4. 振幅（中身）をスワップ
            std::swap(state[index], state[pair_index]);
        }
    }
}

```

### 🔍 コードのポイント解説

* **`if (control_val == 1 && target_val == 0)` による2重スワップの自動ガード**
* Xゲートのときと同様に、`target_val == 0` の時だけ処理するように制限をかけています 。


* もしこの制限がないと、ループが後半に進んで `target_val == 1` のインデックスに到達したときに、すでにスワップした相手と再度スワップしてしまい、元の状態に戻るバグ（2重スワップ）が発生します 。この `if` 文1つで、安全に片方のペアだけを狙い撃ちできます 。




* **ビット和（OR演算：`|`）による `pair_index` の生成**
* 相方のインデックス（`target` 番目のビットが `1` のもの）を作るために、ビット和を使っています。
* `1 << (NUM_QUBITS - 1 - target)` によって「ターゲット位置だけに `1` が立ったマスク」を作成し、現在の `index` と `|`（OR演算）を行うことで、他のビットを一切変えずにターゲット位置だけを確実に `1` に強制書き換えできます。
