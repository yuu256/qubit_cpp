## 1. C++における10進数から2進数への変換方法

* 
**2で割り算を繰り返す方法（アルゴリズムの自作）**

## 2. 量子シミュレータにおける特定のビット値の抽出

量子シミュレータの内部処理（制御ゲートの実装など）では、「あるインデックスの特定のビットの値（0か1）を抽出する」というビット演算が頻出します 。

### 定番のビット演算式

```cpp
// index のなかの「target番目の量子ビット」の値（0か1）を取り出す式
// ※最上位ビットをQ0、最下位ビットをQNとする
int bit_val = (index >> (NUM_QUBITS - 1 - target)) & 1;

```

### 処理の2つのステップ

1. 
**右シフト (`>>`)**：目的のビット（`target`）を、一番右端（最下位ビット）まで移動させます 。


2. 
**論理積 (`& 1`)**：右端に持ってきたビット以外をすべて `0` にクリアし、そのビットが `0` か `1` かを確定させます 。



### 各パーツの詳細解説

* 
**`NUM_QUBITS - 1 - target` （シフト量の計算）** 


* 今回は、一般的なビットの数え方（右端が0番目）とは異なり、「左端（最上位）が0番目（Q0）」というルールを想定しています 。


* そのため、全量子ビット数（`NUM_QUBITS`）から「右端から数えて何桁目にあるか」をこの式で算出します 。


* 
*例*：`NUM_QUBITS = 4` で `target = 1` (Q1) の場合、右端からの距離は $4 - 1 - 1 = 2$ となります 。




* 
**`index >> (...)` （右シフト演算）** 


* 調べたいビットを一番右端の位置まで引きずり出す操作です 。


* 
*例*：`index = 10`（2進数で `1010`）のとき、`target = 1`（左から2番目の `0`）を調べたい場合、右に2回シフト（`>> 2`）して右端に `0` を持ってきます 。




* 
**`& 1` （マスク処理・抽出）** 


* 右シフトした後の値と `1`（2進数で `0001`）の論理積（AND演算）をとります 。


* AND演算は「両方とも 1 のところだけが 1 になる」ため、右端以外の数字は強制的に `0` にクリアされ、目的のビット値だけがピンポイントで抽出できます 。





$$\text{bit\_val} = (\text{index} \gg \text{右端からの距離}) \ \& \ 1$$

## 3. 量子Xゲート（NOTゲート）の正しい認識と実装

量子シミュレータにおいて、「Xゲート＝単なるビット反転」という古典コンピュータの感覚のままだと、コードの実装意図（なぜスワップが必要なのか）が掴めなくなります 。

### 古典コンピュータと量子シミュレータの違い

| 項目 | 古典コンピュータ（NOT） | 量子シミュレータ（Xゲート） |
| --- | --- | --- |
| **保持しているデータ** | メモリの中に `0` または `1` というデータが**直接**入っている 。

 | 0や1そのものではなく、すべての状態（可能性）がどれくらいの確率で存在するかという**「振幅（確率の波）」**を配列（`state`）に保持している 。

 |
| **ゲートの処理内容** | データそのものを直接 `0` から `1`、`1` から `0` へ書き換える 。

 | <br>`target`ビットが `0` である状態の振幅と、`1` である状態の**振幅を丸ごと入れ替える（スワップする）** 。

 |

### 具体例：2量子ビットでターゲットがQ0（左から1番目）の場合

Xゲートを適用すると、すべての状態のQ0ビットが反転するため、確率の波が以下のように移動します 。

* 
`00` という状態の振幅 $\rightarrow$ `10` の位置へ移動 


* 
`01` という状態の振幅 $\rightarrow$ `11` の位置へ移動 



シミュレータの内部視点では、これは `state[00]` と `state[10]`、`state[01]` と `state[11]` の中身を入れ替える **配列要素のスワップ（Swap）処理** になります 。

> **⚠️ 認識のズレに注意**
> * **NGな認識**：`index` 変数自体のビットを直接書き換えて終わりにする（これでは配列の中身の振幅が動かないため誤り） 。
> * **OKな認識**：`index` のビットを操作して「ペアになる相手の部屋番号（インデックス）」を特定し、**配列の要素（振幅）同士を `std::swap` する** 。
> 
> 

### 2重スワップを防ぐための実装アプローチ

0から31まで（5量子ビットの場合）愚直にループを回して、対象を見つけるたびにスワップしてしまうと、「0の項と1の項を入れ替えた後、ループが進んで1の項に到達したときに、また元に戻してしまう」というバグ（2重スワップ）が発生します 。これを防ぐアプローチは主に2つあります。

1. 
**最初から target ビットが0であるインデックスだけを狙ってループを回す** 


* ビット演算を使って、対象となるインデックス（`0`の項）を効率よく作り出す方法です 。無駄なループや条件分岐を減らせるため、シミュレータ実装において非常に推奨されるテクニックです 。




2. 
**0〜31までループを回し、条件分岐を入れる** 


* ループの中で「今のインデックスの target ビットが0のときだけ処理する（1のときはスルーする）」という条件分岐を挟みます 。この判定の際に、前述した「ビット抽出の式」がそのまま役に立ちます 。

# 【C++×量子計算】学習ログ Vol.2：1重ループによるCNOTゲートの実装

C++を用いた量子計算シミュレータの実装を学ぶ過程の学習ログ第2弾です 。前回のVol.1（ビット操作とXゲートの基礎）を踏まえ、今回は2つの量子ビット間で相互作用を行う「CNOTゲート（制御NOTゲート）」の本質的な概念と、シミュレータにおける効率的な実装アプローチについてまとめます 。

---

## 1. CNOTゲートのシミュレータにおける概念

CNOT（Controlled-NOT）ゲートは、2つの量子ビット（**controlビット**と**targetビット**）を対象とするゲートです 。古典計算の `if` 文（条件分岐）に似ていますが、状態ベクトルを扱うシミュレータの内部視点では、やはり「振幅（確率の波）のスワップ処理」として捉える必要があります 。

### 古典計算の直感とシミュレータの処理の対応

* 
**古典的なルール**：「controlビットが `1` の項だけ、targetビットの `0` と `1` を入れ替える」 


* 
**シミュレータ内部の処理**：「**controlビットが `1**`」かつ「**targetビットが `0**`」であるインデックス（部屋番号）を見つけ、それに対応する「targetビットが `1` のインデックス」と**中身（振幅）をスワップする** 



Xゲートがすべての項の `0` と `1` をひっくり返したのに対し、CNOTゲートは「controlビットが `1` である世界（状態）」にだけピンポイントでXゲート（スワップ）を適用する操作になります 。

---

## 2. なぜ「2重ループ」ではなく「1重ループ」で良いのか

「2つのビットをチェックする」と考えると、直感的にループを2重に回したくなりますが、状態ベクトル（`StateVector`）の配列を走査する際は「0〜31（5量子ビットの場合）を巡回する1重ループ」で完全に処理できます 。

1回のループ（ある1つのインデックス `index`）の中で、Vol.1で学んだ「ビット抽出式」を使い、**1つのインデックスから2つのビットの値を同時に取り出す**というアプローチを取るためです 。

### 1回分のループの思考フロー

1. ループで現在の `index` を確認 


2. 
`index` の `control` 番目のビットが `0` か `1` かを調べる 


3. 
`index` の `target` 番目のビットが `0` か `1` かを調べる 


4. 条件（`control == 1` かつ `target == 0`）を満たしていたら、ペアを探してスワップする 



---

## 3. CNOTゲートの具体的なC++実装

TODOの指示（0〜31までループ）通りに、ビット抽出式とビット和を用いて実装した具体的なコード例です 。

```cpp
// CNOTゲート：controlビットが1の項だけ、targetビットの0と1を入れ替える
void apply_CNOT(StateVector& state, int control, int target) {
    // 5量子ビットの場合（状態数は32通り）
    for (int index = 0; index < 32; ++index) {
        // 1. controlビットとtargetビットの値をそれぞれ抽出
        int control_val = (index >> (NUM_QUBITS - 1 - control)) & 1;
        int target_val  = (index >> (NUM_QUBITS - 1 - target)) & 1;
        
        // 2. 「controlが1」かつ「targetが0」の項だけを処理する（2重スワップ防止）
        if (control_val == 1 && target_val == 0) {
            // 3. targetビットを1に反転させたペアのインデックスを計算
            int pair_index = index | (1 << (NUM_QUBITS - 1 - target));
            
            // 4. 振幅（中身）をスワップ
            std::swap(state[index], state[pair_index]);
        }
    }
}

```

### 🔍 コードのポイント解説

* **`if (control_val == 1 && target_val == 0)` による2重スワップの自動ガード**
* Xゲートのときと同様に、`target_val == 0` の時だけ処理するように制限をかけています 。


* もしこの制限がないと、ループが後半に進んで `target_val == 1` のインデックスに到達したときに、すでにスワップした相手と再度スワップしてしまい、元の状態に戻るバグ（2重スワップ）が発生します 。この `if` 文1つで、安全に片方のペアだけを狙い撃ちできます 。




* **ビット和（OR演算：`|`）による `pair_index` の生成**
* 相方のインデックス（`target` 番目のビットが `1` のもの）を作るために、ビット和を使っています。
* `1 << (NUM_QUBITS - 1 - target)` によって「ターゲット位置だけに `1` が立ったマスク」を作成し、現在の `index` と `|`（OR演算）を行うことで、他のビットを一切変えずにターゲット位置だけを確実に `1` に強制書き換えできます。

# 【C++×量子計算】学習ログ Vol.3：量子ビットの測定と状態ベクトルの収縮・規格化

C++を用いた量子計算シミュレータの実装を学ぶ過程の学習ログ第3弾です。Vol.1（Xゲート）、Vol.2（CNOTゲート）では振幅の入れ替え（スワップ）のみを扱いましたが、今回は量子計算における最難関の1つである「量子ビットの測定（収縮と規格化）」を取り上げます 。

確率の波を書き換えて1つの現実にリセットする破壊的な操作を、どのようにC++コードに落とし込むのか、初期案（下書き）からの改善点や数学的なショートカットを交えてまとめました 。

---

## 1. 量子ビット測定の概念とシミュレータでの役割

これまでのゲート操作は、すべての状態の振幅（確率の波）をルールに従ってパズルのように入れ替えるだけでした。しかし「測定」は、それら重なり合っていた可能性を狭め、システムの状態を変化させてしまう**破壊的な操作**です 。

シミュレータの内部視点では、以下のステップを順に実行する必要があります。

1. 対象ビットが `0`（または `1`）である状態の確率をすべて足し合わせる。
2. その確率に基づいて、現実の測定結果（`0` か `1`）をランダムに決定する。
3. 
**【状態の収縮】** 選ばれなかった方の状態（可能性）の振幅をすべて `0` にして消滅させる 。


4. 
**【規格化】** 残った状態の確率の合計が再び $1$（100%）になるように、全体の振幅を引き伸ばして調整する 。



---

## 2. 初期案と改善案の比較：洗練させる4つのTODO

直感的な初期案（下書き）から、シミュレータとして正しく、かつ高速に動作するコードへブラッシュアップするポイントを4つのステップで比較・解説します。

### ① 複素数から確率（絶対値の2乗）への変換

* 
**初期案（直感）**： `pow(state[index], 2.0)` などの単純な累乗計算 。


* 
**改善案（正解）**： `std::norm(state[index])` を使用する 。


* **比較と理由**：
C++の複素数型（`std::complex`）に対して `std::pow` をそのまま適用しようとするとコンパイルエラーの原因になります。また、絶対値を求める `std::abs` を2乗する方法も、内部で平方根（$\sqrt{a^2 + b^2}$）の重い計算が走るため低速です 。`<complex>` ヘッダの **`std::norm(z)`** は直接「絶対値の2乗（$|z|^2$）」を計算してくれるため、確率（振幅の絶対値2乗）を求める際は最もスマートで高速な選択肢となります 。



### ② 確率判定のための乱数生成

* 
**初期案（古典的）**： `rand() % 2` や `rand() / (double)RAND_MAX` などの利用 。


* 
**改善案（現代的）**： `<random>` ヘッダの `std::mt19937` ＋ `std::uniform_real_distribution` 。


* **比較と理由**：
C言語由来の `rand()` は乱数の精度や周期が低く、量子シミュレータが扱うような「$0.4321$ の確率で $0$ になる」といったシビアな確率判定において偏りが発生しやすく不向きです 。現代的なC++では、信頼性の高いメルセンヌ・ツイスタ（`std::mt19937`）と一様実数分布を組み合わせることで、高精度な $0.0 \sim 1.0$ の実数乱数を生成するのがベストです 。



### ③ 状態の収縮（選ばれなかった世界の消滅）

* **処理内容**：
測定によって結果（`result`）が確定した瞬間、もう一方の可能性は完全に消滅します 。コード上では、全インデックスを走査し、**「測定結果と一致しない対象ビット」を持っている部屋の振幅をすべて `0.0` に上書き**することで、この物理的な状態の収縮を忠実に表現できます 。



### ④ 状態ベクトルの規格化（Normalization）

* 
**初期案（直感）**： 「配列の中身を足して、全体をその合計で割る」という単純な加算処理 。あるいは、配列をもう一度再走査して全要素の2乗和の平方根（ノルム）を愚直に計算し直す 。


* 
**改善案（高速化）**： 「規格化のショートカット定理」を利用し、選ばれた確率の平方根で割る 。


* **比較と理由**：
量子力学における規格化は、単純な振幅の足し算ではなく「確率（2乗和）の合計を $1$ にする」作業です 。一部の項を `0` にしたことで合計確率は $1$ 未満に減少していますが、実は「測定で生き残った世界の新しいベクトルの長さ（ノルム）」は、最初に求めた「選ばれた世界の確率の平方根（$\sqrt{P}$）」に必ず一致するという強力な数学的性質があります 。
わざわざもう一度配列をすべてループしてノルムを再計算しなくても、選ばれた世界の確率の平方根（`std::sqrt(chosen_prob)`）を分母として、生き残った項を割り算するだけで一発で規格化が完了します 。



---

## 3. 清書した `measure_qubit` 関数の完全な実装コード

TODOの指示と下書きの意図をベースに、上記の改善点とショートカットをすべて反映させて清書したC++の完全な実装例です 。

```cpp
#include <vector>
#include <complex>
#include <random>
#include <cmath>

using StateVector = std::vector<std::complex<double>>;
const int NUM_QUBITS = 5;

// 1量子ビット測定：確率に基づき0か1を返し、状態ベクトルを収縮・規格化する
int measure_qubit(StateVector& state, int target) {
    double prob_0 = 0.0;
    
    // TODO 1. targetビットが0である全てのインデックスの「振幅の絶対値2乗」を足し合わせる
    for (int index = 0; index < 32; ++index) {
        int bit_val = (index >> (NUM_QUBITS - 1 - target)) & 1;
        if (bit_val == 0) {
            // std::norm で安全かつ高速に確率を取り出す
            prob_0 += std::norm(state[index]);
        }
    }
    
    // TODO 2. 現代的なC++の乱数（0.0～1.0）を生成し、判定する
    static std::random_device rd;
    static std::mt19937 gen(rd()); // メルセンヌ・ツイスタ
    std::uniform_real_distribution<double> dist(0.0, 1.0);
    double rnd = dist(gen);
    
    int result = (rnd < prob_0) ? 0 : 1;
    
    // 選ばれた世界の確率（規格化のショートカット用）
    double chosen_prob = (result == 0) ? prob_0 : (1.0 - prob_0);
    
    // 新しい状態ベクトルのノルム（長さ）は、選ばれた確率の平方根になる
    double new_norm = std::sqrt(chosen_prob);
    
    // TODO 3 & 4. 選ばれなかった方の振幅を0にし、残った項を一発で規格化する
    for (int index = 0; index < 32; ++index) {
        int bit_val = (index >> (NUM_QUBITS - 1 - target)) & 1;
        
        if (bit_val != result) {
            // 選ばれなかった世界を 0 に（状態の収縮）
            state[index] = 0.0;
        } else {
            // 生き残った世界を新しいノルムで割る（規格化）
            state[index] /= new_norm;
        }
    }
    
    return result;
}

```

## 1. ビット反転符号の仕組みと状態遷移

量子エラー訂正（ビット反転符号）では、1つのデータ量子ビット（$Q_0$）の情報を2つのデータ量子ビット（$Q_1, Q_2$）にコピー（符号化）して「多数決」が取れる状態を作ります 。さらに、重ね合わせを壊さないように補助ビット（$A_0, A_1$）を間接的に測定してエラーの場所（シンドローム）を特定します 。

シミュレータ内部では、状態ベクトル（配列のインデックス）が以下のように遷移していきます 。今回は量子ビットのインデックス配置を `Q0 Q1 Q2 A0 A1` の順（左が上位ビット）とします 。

1. **初期状態の設定**
データビット $Q_0$ に未知の量子状態をセットします（補助ビットは `0` 固定） 。



$$\lvert \psi \rangle = \alpha \lvert 00000 \rangle + \beta \lvert 10000 \rangle$$


配列上では `state[0]` に $\alpha$、`state[16]` に $\beta$ が入ります 。


2. **符号化（Encoding）**


$Q_0$ をコントロールにして $Q_1, Q_2$ へCNOTをかけることで、以下の絡み合い状態（GHZ状態）を作ります 。



$$\alpha \lvert 00000 \rangle + \beta \lvert 11100 \rangle$$


この時、$\beta$ の振幅は `state[16]` から `state[28]`（$\lvert 11100 \rangle$）へ引っ越します 。


3. **エラー発生（Noise）とシンドローム測定**
データビットのどこか1つ（今回は $Q_1$）にビット反転エラー（Xゲート）が発生したと仮定します 。
状態は $\alpha \lvert 01000 \rangle + \beta \lvert 10100 \rangle$ へと変化します 。
その後、補助ビット $A_0$ に $Q_0 \oplus Q_1$、$A_1$ に $Q_1 \oplus Q_2$ のパリティ情報を書き込み、$A_0$ と $A_1$ を測定します 。データを直接測定しないため、重ね合わせを壊さずにエラー位置を特定できます 。

## 2. 初期案と改善案の比較：陥りがちな2つの罠

下書き段階の初期案から、シミュレータを正しく動作させるために修正した2つのポイントを比較します 。

### 罠①：ストーリー（エラー発生箇所とシンドローム）の矛盾

* 
**初期案**： `apply_X(state, Q0);` （$Q_0$ にエラーを発生させている） 


* 
**改善案**： `apply_X(state, Q1);` （$Q_1$ にエラーを発生させる） 


* **比較と理由**：
初期案のコードでは $Q_0$ にエラーを適用していましたが、その後のシンドローム判定（`if` 文）やコメントでは「$Q_1$ のエラーを検知して直す流れ（`m1==1 && m2==1`）」を想定していました 。これではストーリーが矛盾し、エラー訂正が正常に機能しません 。エラー発生箇所を $Q_1$ に統一することで、シンドローム測定結果が $(m_1, m_2) = (1, 1)$ となり、綺麗に $Q_1$ が救済される整合性を確保しました 。



### 罠②：【重要】デコード（検証）時における CNOT の向き

* 
**初期案**： `apply_CNOT(state, Q1, Q0);` （$Q_1$ がコントロール、$Q_0$ がターゲット） 


* 
**改善案**： `apply_CNOT(state, Q0, Q1);` （$Q_0$ がコントロール、$Q_1$ がターゲット） 


* **比較と理由**：
符号化（Encoding）の際、私たちは「$Q_0$ を親（コントロールビット）」として $Q_1, Q_2$ へ状態をコピーしました 。したがって、エラーを直した後にデコード（逆操作）をして元の $Q_0$ に状態を一本化するときも、「$Q_0$ を親（コントロールビット）」としてCNOTをかけるのが正解です 。
もし初期案のように向きを逆にしてしまうと、$\alpha \lvert 00000 \rangle + \beta \lvert 11100 \rangle$ の状態から $\beta$ 側の振幅が元の `state[16]` ($\lvert 10000 \rangle$) に綺麗に戻らず、別のインデックスに迷子になってしまいます 。

## 3. 測定後の落とし穴：出力が `(0,0)` になる物理現象の怪

正しくエラー訂正とデコードまで終えた後、検証のために `state[0]` と `state[16]` を出力すると、値が `(0,0)` になってしまい「バグったのか？」と錯覚する現象にぶつかります 。

> **💡 原因はバグではなく、物理現象（状態の収縮）の完璧な再現**
> 今回のエラーシナリオ（$Q_1$ のエラー）では、シンドローム測定の結果、補助ビット $A_0$ と $A_1$ は両方とも `1` に確定します（`m1 = 1, m2 = 1`） 。
> Vol.3 で実装した通り、測定関数 `measure_qubit` は「選ばれなかった方の振幅をすべて `0` にする」という処理を行います 。
> 
> 
> ビット配置が `Q0 Q1 Q2 A0 A1` であるため、補助ビットが `11` に確定した瞬間、**末尾2ビットが `11` ではない項（＝補助ビットが `00` などの項）の振幅はすべて消滅して `0` になります** 。
> * `state[0]` （2進数 `00000`） $\rightarrow$ 補助ビットが `00` なので消滅（`0` に収縮） * `state[16]`（2進数 `10000`） $\rightarrow$ 補助ビットが `00` なので消滅（`0` に収縮） 
> 
> 

### 元のデータ（$\alpha, \beta$）はどこへ消えた？

消えたわけではなく、**補助ビットが `11` になった部屋へ引っ越しています** 。2進数の末尾に `11` がつくということは、10進数でいうと `+3` された位置にいるということです 。

* 
$\alpha$ の引っ越し先： `state[0 + 3]` $\rightarrow$ **`state[3]`** （`00011`） 


* 
$\beta$ の引っ越し先： `state[16 + 3]` $\rightarrow$ **`state[19]`** （`10111`） 



そのため、検証用の出力部分では「測定結果（`m1, m2`）に応じた正しいインデックス」を計算して覗いてあげる必要があります 。

## 4. 清書したメイン処理の完全なC++コード

上記の矛盾やデコードのバグをすべて修正し、コンソールの文字化け対策として日本語メッセージを英数字（英語）に変更した完全なメイン処理のコードです 。

```cpp
#include <iostream>
#include <vector>
#include <complex>

using Complex = std::complex<double>;
using StateVector = std::vector<Complex>;

// 量子ビットのインデックス定義 (左端がQ0)
const int Q0 = 0;
const int Q1 = 1;
const int Q2 = 2;
const int A0 = 3;
const int A1 = 4;

const int NUM_QUBITS = 5;
const int STATE_SIZE = 1 << NUM_QUBITS; // 32

int main() {
    // 状態ベクトルの初期化（全要素0）
    StateVector state(STATE_SIZE, 0.0);
    
    // 1. 初期状態の設定 (|ψ> = alpha|00000> + beta|10000>)
    double alpha = 0.6;
    double beta = 0.8;
    state[0] = alpha;  // |00000>
    state[16] = beta; // |10000> (1 << 4)
    
    std::cout << "--- 3-Qubit Bit-Flip Code Simulation ---" << std::endl;
    
    // 2. 符号化 (Encoding) : Q0 の状態を Q1, Q2 にコピー
    apply_CNOT(state, Q0, Q1);
    apply_CNOT(state, Q0, Q2);
    
    // 3. エラー発生 (Noise) : 今回は Q1 にビット反転エラーが発生したと仮定
    apply_X(state, Q1); 
    
    // 4. シンドローム測定の仕込み : 補助ビット A0, A1 にパリティを書き込む
    apply_CNOT(state, Q0, A0);
    apply_CNOT(state, Q1, A0);
    apply_CNOT(state, Q1, A1);
    apply_CNOT(state, Q2, A1);
    
    // 5. 測定の実行 : 補助ビット A0, A1 の測定結果(m1, m2)を取得
    int m1 = measure_qubit(state, A0); // Q1エラーの場合 = 1
    int m2 = measure_qubit(state, A1); // Q1エラーの場合 = 1
    std::cout << "Syndrome Measured: (m1, m2) = (" << m1 << ", " << m2 << ")" << std::endl;
    
    // 6. 復元 (Correction) : 古典ビットの測定結果に応じて if 文で分岐し、エラーを訂正
    if (m1 == 1 && m2 == 0) {
        apply_X(state, Q0);
    }
    else if (m1 == 1 && m2 == 1) {
        apply_X(state, Q1); // 見事 Q1 のエラーが救済される
    }
    else if (m1 == 0 && m2 == 1) {
        apply_X(state, Q2);
    }
    
    // 7. 検証 (Decoding) : 符号化の逆操作（Q0を親にしてCNOT）
    apply_CNOT(state, Q0, Q1);
    apply_CNOT(state, Q0, Q2);
    
    // 8. 結果の出力 : 状態収縮によって移動した先のインデックスを計算して確認
    int final_base_index = (m1 << 1) | m2; // 測定結果から補助ビットのズレ分を計算
    int idx_alpha = 0 + final_base_index;  // alphaの生存している部屋 (Q0=0)
    int idx_beta  = 16 + final_base_index; // betaの生存している部屋  (Q0=1)
    
    std::cout << "--- Final Result Verification ---" << std::endl;
    std::cout << "state[" << idx_alpha << "] (alpha): " << state[idx_alpha] << " (Expected: 0.6)" << std::endl;
    std::cout << "state[" << idx_beta << "] (beta) : " << state[idx_beta] << " (Expected: 0.8)" << std::endl;
    
    return 0;
}

```
